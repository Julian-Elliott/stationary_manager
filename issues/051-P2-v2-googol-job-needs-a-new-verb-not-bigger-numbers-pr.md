# v2 GOOGOL job needs a new verb, not bigger numbers — prototype the sorting mechanic before building the planet

**Priority:** P2  
**Filed by:** solodev_ren (solo indie dev, ships mobile games)  
**Labels:** v2, design, game-feel

---

The recruitment letter promises Director of E-Mail at 4.1 billion paper letters/day. The trap — and I have fallen into this trap personally, on stream, twice — is shipping v2 as v1 with more zeroes: bigger requests, faster trucks, same order → unbox → drag-to-desk loop. That's a reskin and players smell it in five minutes.

**What v1's verb is:** point-to-point logistics. Order, burst boxes, drag bin to desk. It's great at office scale and it mathematically cannot be the verb at 4.1 billion/day — one thumb cannot drag four billion letters, and the game KNOWS that, because the whole hiring joke ('humans doing what one Excel macro did') is about the absurdity of manual throughput.

**What the GOOGOL verb should be:** routing, not carrying. You stop touching items and start touching the SYSTEM — flipping switch-tracks on mail chutes, stamping batch triage rules ('everything addressed to hyperspace goes LEFT'), placing human sorters (the runners economy, matured) and fixing the jams they cause. The player's finger moves from objects to infrastructure. That's a real sequel jump, same fiction, same narrator, new muscle.

**Concrete asks:**
1. Before any v2 world-building: a one-file prototype of JUST the sorting verb with placeholder art, 10-minute playtest loop. If routing letters isn't fun grey-boxed, no amount of GOOGOL office art saves it.
2. Decide the narrator's v2 relationship now, because v1's finale sets it up: he reads your recruitment letter with barely disguised jealousy. Is he your subordinate at GOOGOL? Passed over for the job? That tension is the sequel's spine and v1's final barks should foreshadow whichever we pick.
3. Numbers at that scale need log-scale presentation — the odometer gag survives (a 12-digit odometer is FUNNIER) but request UI ('deliver 40,000,000 letters') needs a progress-table treatment like Day 6's TPS RECONCILIATION, which is quietly the v2 UI prototype already sitting in v1. Note that in the code comments so we don't rediscover it.
4. Carry-forward hook: the personnel file (MBTI diagnoses, compliance dial history) should be v2's import. Cheap to spec now, impossible to retrofit.
